Tool Classification Guide

Every tool call in this presentation has been validated against official vendor MCP repositories. Each is classified as follows:

Official Official MCP Tool

Tool exists in the vendor's published MCP server with the exact (or functionally equivalent) name. Verified against official GitHub repos.

Sources: Log360 Cloud Planned APIs V1, CrowdStrike falcon-mcp, Okta okta-mcp-server, GitHub github-mcp-server, SentinelOne purple-mcp, AWS cloudtrail-mcp-server

REST API Custom Tool (REST API Integration)

No official MCP tool exists. Would need to be built as a custom MCP tool wrapping the vendor's REST API. The capability exists in the vendor's API but is not exposed via their MCP server.

Examples: CrowdStrike endpoint isolation, Okta session revocation, Palo Alto IP blocking, M365 email investigation, AWS IAM key management

Community Custom Tool (Community)

Tool exists in a community-maintained MCP server (not official vendor). May have different tool names or limited functionality.

Example: Softeria ms-365-mcp-server (productivity focus, not security investigation)

Custom Custom Tool

Vendor has no MCP server at all. Would need to be built entirely as a custom MCP server wrapping their API.

Examples: ServiceNow, Qualys, VirusTotal

⚠️ Note: CrowdStrike falcon-mcp, Okta okta-mcp-server, and SentinelOne purple-mcp are read-only MCP servers. Write/action operations (isolate, block, revoke) require custom REST API integration even when the vendor has an official MCP server.

What Happened?

T+0: Developer Ajay installs trojanized Python package from a look-alike domain (registered 1 week ago)
VS Code → python.exe → malicious_script.py
T+2 min: Script reads plaintext AWS credentials from ~/.aws/credentials — a common but dangerous developer practice
T+5 min: Runs AWS reconnaissance (sts get-caller-identity) → lists S3 buckets → creates new access key for svc-etl-prod (production service account)
T+15 min → T+90 min: Downloads 18 GB via 2,400 S3 GetObject calls — 14.2 GB customer PII + 3.8 GB financial reports
T+90 min: Data exfiltrated to external server linked to cloud-targeting threat infrastructure
Root cause: Plaintext AWS credentials on dev workstation + unvetted package install. Under 2 hours from install to breach.
